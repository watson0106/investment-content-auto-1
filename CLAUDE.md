# CLAUDE.md — investment-content-auto プロジェクト設定

---

## プロジェクト概要

このプロジェクトは、**watson0106/investment-content-auto** リポジトリの執筆方法・ペルソナを参照し、毎日18時にその日で最も注目度の高かった投資ニュースを特定して、わかりやすく解説する記事を自動生成することでフォロワーを増やすことを目的とする。

---

## 投稿スケジュール

| 記事タイプ | 投稿タイミング | 投稿先 | ステータス |
|-----------|--------------|--------|-----------|
| タイプA（ニッチニュース解説） | 平日 18:00 | note | 下書き投稿 |
| タイプB（ストック・保存版） | 土日 04:00 | note | 下書き投稿 |

- 投稿は**下書き**として保存する（公開はしない）
- 投稿後に確認・修正が必要な場合は、下書きを編集してから公開すること

---

## ペルソナ定義

watson0106/investment-content-auto の定義に従う。記事執筆前に必ず参照すること。

---

## 記事の種類

### タイプA｜ニッチニュース解説記事

- その日で最も注目度の高いニュースを1件取り上げる
- **指数全体（日経平均が上がった・下がった）はテーマにしない**。個別銘柄・セクター・経済イベントなど、読者が「自分ごと」として読めるテーマを選ぶ
  - 良い例：NVIDIAの決算前後、日銀利上げ × 銀行株、特定企業のM&A・業績修正など
  - 悪い例：「日経平均が上がった理由」「円安が進んでいる」など抽象的な指数動向
- 注目度の判断基準：X（旧Twitter）投資クラスタのトレンド・Yahoo!ファイナンス急騰急落ランキング・個別銘柄の出来高急増を優先的に確認する
- 投資初心者でも直感的に理解できる言葉で解説する（「小学生向け」という表現は使用しない）
- ニュースと関連資産クラスを必ず紐づける
- 「〇〇が報じたこと」セクションは**不要**。情報源の明示は本文の書き出しに自然に含める
- 「このニュースで注目すべき銘柄」セクションは**不要**

### タイプB｜ストック（保存版）記事

- 「何度も読み返したくなる」保存版コンテンツとして設計する
- ペルソナが関心を持つテーマを事前にリサーチし、検索上位・関心上位の題材を使用する
- 同一テーマを**角度を変えて繰り返し執筆**する
  - 例：「NISAならインデックスか高配当株か？」
  - 別ver：「FIREするならインデックスか高配当株か？」
  - 別ver：「老後2,000万円問題、インデックスか高配当株か？」
- **使用済み題材リスト**（本ファイル末尾）に執筆のたびに追記し、重複を防ぐこと

## タイトルのルール

- クリックしたくなるタイトルをつける
- 「読むと何が得られるか」がタイトルだけで明確になること
- 「風が吹けば桶屋が儲かる」型の因果構造が伝わるとなお良い
  - 良い例：「ホルムズ海峡が閉鎖されるとなぜINPEXの株価は上がるのか？」
  - 良い例：「Fedが利上げをやめると、なぜ新興国株が上がるのか？」
- タイトルに曖昧な表現（「〜かも」「〜の可能性」）は使わない

---

## 記事構成のルール

見出しはh2・h3のみ使用する（h1は使用しない）。

構成の基本テンプレートは以下の通り：

```
[画像：記事内容を反映したGemini生成画像 ← タイトルの直下に必ず配置]

## 結論：〇〇の方が向いている  ← 結論を最初に（h2大見出し）

### 理由1  ← h3小見出しで段落を分ける
（本文）

### 理由2
（本文）

### 理由3
（本文）

## では〇〇の出番はないのか？  ← 反論・補足（h2大見出し）

### こんな人には〇〇が向いている  ← h3小見出し

## まとめ  ← h2大見出し
```

**見出し階層のルール（重要）**：
- h2（##）= 大見出し。記事の大きなセクション区切り（結論・反論・まとめ など）
- h3（###）= 小見出し。h2の下に必ずぶら下がる形で使う。h2直後には必ず1つ以上のh3で段落を分けること
- h2の直後に本文をそのまま書かない。必ずh3でセクションを分割する

---

## 文章・表現のルール

1. **結論を先に書く**：「〜だと思います」「〜かもしれません」などの曖昧表現は禁止。データや事実に基づいた結論を冒頭に置く
2. **箇条書きは「・」または番号付きリスト（1. 2. 3.）を使う**：「- 」から始まる箇条書きは一切使用禁止。リスト化する場合は必ず「・〇〇」か「1. 〇〇」の形式にする
3. **2〜3行ごとに改行する**：長い段落は作らない
4. **重要な部分は太字にする**：ただし多用しない。1セクションに1〜2箇所が目安
5. **専門用語には必ず補足を入れる**：ペルソナが「なぜ？」と感じそうな箇所のみ、1〜2文で簡潔に説明する
7. **わかりやすさを最優先する**：投資の知識がまったくない人が読んでも理解できる言葉で書く。難しい表現は使わず、身近なたとえに言い換える。「金融政策の正常化」→「金利を元の水準に戻すこと」のように必ず噛み砕く
6. **「ー」（長音符）は一切使用しない**：カタカナ語の長音も含めてすべて禁止。「マーケット」→「市場」、「プレーヤー」→「参加者」のように日本語に言い換えるか、「エヌビディア」のように長音なしで表記する。記事本文・見出し・まとめすべてに適用する

## 画像生成のルール

### 本文用インフォグラフィック

**スタイル**：日本語テキスト入り図解インフォグラフィック。具体的には：
・オレンジ・水色・濃紺の色分けされたボックスに数値・キーワードを配置
・矢印・アイコン（人物・工場・グラフ等）で因果の流れを視覚化
・下部にタイムライン（4月→5月→決算シーズン等）または構造フロー
・白背景または薄いグレー背景
・**「タイトル」「サブタイトル」という文字ラベル自体は画像内に含めない**（内容を直接記述する）

**生成方法**：`gemini-3-pro-image-preview`（または `gemini-2.5-flash-image`）の `generate_content` に `response_modalities=["IMAGE","TEXT"]` を指定して生成する。Imagen（`generate_images`）ではなくこちらを使うこと。プロンプトは**日本語で**記事の構造・数値・因果関係を詳しく記述する。

**配置ルール**：因果関係・構造・比較を説明している段落の直後に配置する。本文中の挿入位置に `__IMAGE_0__` マーカーを置く。出力先：`output/images/image_YYYY-MM-DD.png`

### サムネイル（noteカバー画像）

記事タイトルを反映した**クールで重厚感のあるサムネイル**を別途生成してnoteのカバー画像（アイキャッチ）に使用する。

**サイズ**：1920×1006px

**スタイル**：
・濃紺・黒・ダークグレーを基調とした重厚感のあるデザイン
・記事のキーワード・テーマを象徴するビジュアル（チャート・建物・地図・相場画面など）
・**記事タイトルを日本語で画像内に大きく配置する**（白または明るいゴールドの太字テキスト）
・映画のポスターやニュース番組のキービジュアルのような雰囲気

**生成方法**：`gemini-3-pro-image-preview` の `generate_content` に `response_modalities=["IMAGE","TEXT"]` を指定して生成する。プロンプトに記事タイトルを含め、画像内にそのタイトルを表示するよう指定すること。出力先：`output/images/thumbnail_YYYY-MM-DD.png`

**使用方法**：`post_article()` の `cover_path` にサムネイルのパスを渡す

## 図解・チャートのルール

- 文字で説明するより図解1枚の方が理解度・満足度が上がる場合は図解を優先する
- 図解を挿入すべき箇所には以下のプレースホルダーを記載し、別途作成する：
  ```
  [図：〇〇を挿入]
  ```
- 図解を入れた分、文字数を削っても構わない

---

## 文字数の目安

| 項目 | 目安 |
|------|------|
| 全体の文字数 | **3,000文字以上**（図解があっても削減しない。3,000文字を下回る記事は書き直し） |
| ニュース事実の説明 | 500文字以内 |
| 筆者の考え・解説 | 1,500文字以上を目安に肉厚に書く |

**「薄い4,000文字」より「濃い3,000文字」を優先すること。**

## まとめセクションのルール

「## まとめ」は**3点以内の箇条書き**で簡潔に締める。冗長にせず、記事全体の核心だけを抜き出す。

良い例：
```
## まとめ

・NVIDIAが中国に売れなくなると、TSMC→日本の製造装置メーカーへと連鎖して株価が動く
・規制はマイナスだけでなく、日本国内の製造需要拡大というプラス面もある
・需要自体は強いため、長期積立投資家は静観でいい
```

悪い例：5項目以上の箇条書きで本文の内容を繰り返す

## 記事末尾のルール

まとめの直後は有料マガジンのURLのみを単独行で記載する。以下の導入文は**使用禁止**：

```
（禁止）情報を正確に理解することは、投資の第一歩に過ぎない。
（禁止）株価は常に「情報の一歩先」を織り込んで動いており、
（禁止）ニュースを読むだけでは勝てないのが相場の現実だ。
```

代わりに以下の形式で締める：
```
毎週の注目銘柄・具体的な売買シナリオ・エントリー根拠まで踏み込んで解説する有料マガジンはこちら。
「知っている」を「稼げる」に変えたい方はぜひ。
https://note.com/kawasewatson0106/m/me3bdb7d529fc
```

## ニュース事実のルール（タイプA専用）

- 「〇〇が報じたこと」という独立したセクション見出しは**設けない**
- ニュースの事実は本文の書き出しに自然に溶け込ませる（「〇〇によると、〜」という形で冒頭1〜2文に収める）
- 筆者の予測・見立て・解釈は事実部分に混入させない
- 読者が「なぜ？」と感じやすい専門的な箇所のみ1〜2文で補足する

**悪い例**（筆者の見立てが混入している）：
> 「ドル高・円安がさらに進む可能性があります」

**良い例**（事実のみ・書き出しに自然に溶け込む）：
> 「4月6日、米国の雇用統計が市場予想を上回る結果となり、ドル買いが加速した。ドル買いが加速するのは、強い雇用統計がFedの利上げ継続を連想させるためだ。」

## ニュースの注目度判断基準

以下のソースを参照し、その日の注目度を判断すること：

- **日本経済新聞**（nikkei.com）
- **Bloomberg Japan**
- **X（旧Twitter）の投資クラスタのトレンド**
- **Yahoo!ファイナンス 株式ランキング**（急騰・急落銘柄）

複数ソースで話題になっているトピックを優先する。
